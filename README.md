# Прогноз конверсии пользователей по данным веб‑аналитики

Проект решает задачу прогнозирования конверсии сессии пользователя на сайте по данным веб‑аналитики  
Модель по сессионным признакам оценивает вероятность целевого действия

---

## Содержимое репозитория

Структура проекта:

- `data/`
  - `ga_hits.csv` – сырые данные по хитам (просмотры страниц, события и т.п.).
  - `ga_sessions.pkl` – подготовленные сессионные данные после агрегации.
  - `sample_infer.csv` – пример входного файла для инференса.
  - `predictions.csv` – пример файла с предсказаниями модели.
- `models/`
  - `model.pkl` – сериализованный пайплайн с моделью. (из-за ограничения в 25МБ не влез в гит)
- `src/`
  - `infer.py` – скрипт инференса: загрузка модели, чтение входных данных, запись предсказаний.
  - (при наличии) вспомогательные модули: подготовка признаков, обучение, метрики.
- `hakaton_answer.ipynb` – ноутбук с разведочным анализом данных, визуализациями и экспериментами с моделями.
- `README.md` – текущее описание проекта.

---

## Постановка задачи

**Цель:** по информации о сессии предсказать, будет ли достигнута целевая конверсия (`is_target`).

**Объект моделирования:**

- Сессия пользователя на сайте (визит).
- Для каждой сессии доступны как сырые хит‑данные, так и агрегированные признаки.

**Основные задачи:**

- Провести EDA: понять поведение пользователей, источники трафика и распределения признаков.
- Построить и сравнить несколько моделей (логистическая регрессия, случайный лес и др.).
- Оценить качество (ROC‑AUC) и выбрать итоговую модель.
- Собрать удобный скрипт инференса, который:
  - принимает CSV с описанием сессий,
  - возвращает вероятность конверсии для каждой строки.

## Данные и признаки

### Исходные данные

- `ga_hits.csv`  
  Хит‑уровень: события внутри сессий (страницы, клики, события), используется для агрегаций и построения дополнительных фич.

- `ga_sessions.pkl`  
  Сессионный уровень: каждая строка – одна сессия с уже собранными базовыми признаками и целевой переменной.

### Целевая переменная

- `is_target` – бинарный флаг конверсии:
  - `0` – нет целевого действия,
  - `1` – есть целевое действие (конверсия).

### Основные признаки

Базовые:

- `visit_number` – номер визита пользователя.
- `visit_month` – месяц визита.
- `visit_dayofweek` – день недели визита.
- `visit_hour` – час начала сессии.
- `is_paid` – признак платного трафика.
- `is_social` – признак трафика из соцсетей.

Категориальные (используются и как отдельные фичи, и для анализа качества трафика):

- `city` – город пользователя.
- `source` – источник трафика (yandex, google, direct, social, email и т.д.).
- `device_category` – тип устройства (desktop, mobile, tablet).

Дополнительно из EDA и расширенного анализа:

- Производные признаки по типу трафика, устройству, городу и др. (см. ноутбук `eda.ipynb`).

## Анализ данных и визуализации

В ноутбуке проведён подробный EDA, включая:

1. **Баланс классов (конверсии):**
   - Столбчатая диаграмма и круговая диаграмма долей классов.
   - Подписи с процентами для наглядной иллюстрации дисбаланса.

2. **Конверсия по источнику трафика:**
   - Горизонтальные bar‑чарты конверсии по сегментам:
     - Органика (`is_organic`),
     - Платный трафик (`is_paid`),
     - Соцсети (`is_social`).
   - Для каждого типа трафика рассчитывается CR в процентах и отображается в подписях на графике.

3. **Корреляционный анализ признаков:**
   - Матрица корреляций Пирсона для числовых признаков.
   - PhiK‑матрицы для оценки взаимосвязи категориальных признаков с целевой:
     - Отдельно для базовых признаков.
     - Отдельно для расширенных признаков.

4. **Важность признаков:**
   - Для случайного леса – важности признаков (`feature_importances_`) с топ‑20 самых значимых фич.
   - Для логистической регрессии – топ‑20 весов по абсолютной величине коэффициентов с цветовой индикацией знака (положительное/отрицательное влияние на конверсию).

## Модели и метрики

В работе рассматриваются две основные модели:

### 1. Логистическая регрессия

Подготовка признаков:

- One‑Hot / dummy‑кодирование категориальных признаков:
  - `visit_month`,
  - `visit_dayofweek`,
  - `device_category`,
  - `source`,
  - `city`.
- `pd.get_dummies(..., drop_first=True)` для предотвращения мультиколлинеарности.
- Приведение тестового набора к тем же колонкам через `reindex(..., fill_value=0)`.
- Масштабирование числовых признаков через `StandardScaler`.

Модель:

- `LogisticRegression(class_weight='balanced', max_iter=1000, random_state=42)`.
- Используется взвешивание классов для борьбы с дисбалансом.

Оценка:

- ROC‑кривая (TPR vs FPR).
- ROC‑AUC (`roc_auc_lr`), выводится в легенде и в текстовом формате.

### 2. Случайный лес (Random Forest)

Подготовка признаков:

- Категориальные признаки:
  - `visit_month`,
  - `visit_dayofweek`,
  - `device_category`,
  - `source`,
  - `city`.
- Кодирование через `OrdinalEncoder` с:
  - `handle_unknown='use_encoded_value'`,
  - `unknown_value=-1` – устойчивость к новым категориям на инференсе.

Модель:

- `RandomForestClassifier(n_estimators=100, max_depth=12, class_weight='balanced', n_jobs=-1, random_state=42)`.

Оценка:

- ROC‑кривая и ROC‑AUC (`roc_auc_rf`).
- Анализ важностей признаков (топ‑20 по `feature_importances_`).

## Скрипт инференса

Инференс реализован в `src/infer.py`.

### Ожидаемый формат входных данных

Скрипт читает CSV, совместимый с признаками, на которых обучалась модель. Минимальный набор колонок (может отличаться, если добавлялись новые фичи – см. актуальный код):

- `visit_number`
- `visit_month`
- `visit_dayofweek`
- `visit_hour`
- `is_paid`
- `is_social`
- `city`
- `source`
- `device_category`

Пример входного файла – `data/sample_infer.csv`.

### Логика работы `infer.py`

1. Чтение аргументов командной строки:
   - `--model` – путь к сохранённой модели/пайплайну.
   - `--input` – путь к CSV с данными для инференса.
   - `--output` – путь к файлу, куда сохранятся предсказания.

2. Загрузка модели:
   - Модель хранится как пайплайн, внутри которого:
     - предобработка (кодирование категориальных признаков),
     - сам алгоритм (логистическая регрессия или случайный лес).

3. Чтение входного CSV в `pandas.DataFrame`.

4. Прогон данных через модель:
   - Используется `predict_proba(X)[:, 1]` для получения вероятности конверсии.

5. Сохранение результата:
   - В `data/predictions.csv` или путь, переданный через `--output`.
   - Обычно добавляется колонка `proba` (или аналогичное название) с вероятностью конверсии.

## Как запустить проект

### 1. Установка зависимостей

Создай окружение и установи необходимые библиотеки (список можно вынести в `requirements.txt`):

```bash
pip install -r requirements.txt
```

Минимальный набор:

- `pandas`
- `numpy`
- `scikit-learn`
- `matplotlib`
- `seaborn`
- (при использовании) `phik`

### 2. Подготовка данных

Если у тебя только сырые `ga_hits.csv` и/или другие файлы, нужно:

- Либо воспользоваться уже подготовленным `ga_sessions.pkl`.
- Либо запустить ноутбук `hakaton_answer.ipynb` / соответствующий скрипт подготовки, чтобы собрать сессионный датасет с нужными признаками.

## 3. Обучение модели

В ноутбуке `hakaton_answer.ipynb` последовательно выполняются шаги:

1. Загрузка подготовленного датасета сессий (`data/ga_sessions.pkl`).
2. Разделение данных на обучающую и тестовую выборки.
3. Формирование признаков:
   - категориальные: месяц визита, день недели, тип устройства, источник трафика, город;
   - числовые: номер визита, признаки активности пользователя и др.
4. Обучение нескольких моделей:
   - логистическая регрессия (базовый интерпретируемый бейзлайн);
   - случайный лес;
   - CatBoost для табличных данных с категориальными признаками.
5. Оценка качества по метрике ROC-AUC и выбор финальной модели.
6. Обучение финальной модели на всех данных и сохранение артефакта в `models/model.pkl`.

Итоговые значения ROC-AUC на тестовой выборке:
- логистическая регрессия: ≈ 0.654;
- случайный лес: ≈ 0.684;
- CatBoost: ≈ 0.687 (лучшая метрика среди рассмотренных моделей).

## 4. Инференс

После обучения финальная модель сохраняется в файл `models/model.pkl`.  
Пример запуска инференса из корня проекта:

```bash
python src/infer.py \
  --model models/model.pkl \
  --input data/sample_infer.csv \
  --output data/predictions.csv
```

Скрипт:
- загружает модель и порог отсечения для признака `visit_number`;
- применяет те же преобразования (обрезка `visit_number` сверху до сохранённого порога);
- считает вероятность целевого действия для каждой сессии и сохраняет результат в `data/predictions.csv`.

Проверить результат можно, например, так:

```python
import pandas as pd

preds = pd.read_csv("data/predictions.csv")
preds.head()
```

## 5. Интерпретация результатов

Каждая строка в `predictions.csv` соответствует одной сессии из входного файла (`sample_infer.csv` или вашего датасета).  
Основной выходной столбец — `proba`, вероятность целевого действия в интервале от 0 до 1:

- значения ближе к 1 означают высокую вероятность конверсии;
- значения ближе к 0 — низкую.

Рекомендуемые сценарии использования:
- ранжирование трафика по качеству;
- приоритизация лидов и сегментов для доработки;
- настройка bid‑стратегий и рекламных кампаний с учётом вероятности конверсии.
